# Figure-MCP 시스템 아키텍처

## 개요

Figure-MCP는 개발 표준 파일을 관리하고, 이를 기반으로 요구사항에 맞는 개발 산출물을 자동 생성하는 MCP(Model Context Protocol) 기반 시스템입니다.

## 시스템 목표

- 개발 표준 문서의 중앙 집중식 관리
- 사이트별 개발 표준에 따른 맞춤형 산출물 생성
- Jira 연동을 통한 요구사항 자동 수집
- RAG(Retrieval-Augmented Generation) 기반 지능형 산출물 생성
- 개발자 친화적 인터페이스 제공

## 핵심 컴포넌트

### 1. MCP Server (Core)

**역할**: 시스템의 중앙 제어 및 MCP 프로토콜 처리

**주요 기능**:
- MCP 프로토콜 구현 및 클라이언트 통신
- 명령어 처리: `use design figure {sitename} {jira-ticketId}`
- 작업 플로우 오케스트레이션
- 사용자 세션 관리

**기술 스택**:
- Node.js/TypeScript 또는 Python
- MCP SDK
- WebSocket 또는 HTTP 기반 통신

### 2. Vector Database

**역할**: 개발 표준 문서의 벡터화된 저장소

**주요 기능**:
- 개발 표준 문서의 임베딩 저장
- 사이트별 문서 분류 및 관리
- 유사도 검색 및 컨텍스트 검색
- 버전 관리

**지원 파일 형식**:
- XLSX (Excel 문서)
- DOCX (Word 문서)
- Markdown (MD)
- Plain Text
- PDF

**기술 스택**:
- Chroma, Pinecone, 또는 Weaviate
- 임베딩 모델: OpenAI text-embedding-ada-002 또는 로컬 모델

### 3. Document Processor

**역할**: 다양한 형식의 문서를 처리하고 벡터화

**주요 기능**:
- 다중 파일 형식 파싱
- 텍스트 추출 및 전처리
- 청킹(chunking) 및 임베딩 생성
- 메타데이터 추출 및 관리

**기술 스택**:
- Python: pandas, python-docx, openpyxl
- 텍스트 처리: spaCy, NLTK
- PDF 처리: PyPDF2, pdfplumber

### 4. RAG Engine

**역할**: 검색 증강 생성을 통한 지능형 산출물 생성

**주요 기능**:
- 요구사항 기반 관련 표준 검색
- 컨텍스트 기반 프롬프트 생성
- LLM을 통한 산출물 생성
- 생성 결과 후처리

**기술 스택**:
- LangChain 또는 LlamaIndex
- OpenAI GPT-4, Claude, 또는 로컬 LLM
- 프롬프트 엔지니어링 도구

### 5. Jira Integration

**역할**: Jira 티켓에서 요구사항 자동 추출

**주요 기능**:
- Jira API 연동
- 티켓 정보 추출 (제목, 설명, 요구사항)
- 요구사항 구조화 및 분석
- 우선순위 및 레이블 처리

**기술 스택**:
- Jira REST API
- OAuth 2.0 또는 Personal Access Token
- 요구사항 파싱 로직

### 6. Web UI Dashboard

**역할**: 사용자 인터페이스 및 진행상태 모니터링

**주요 기능**:
- 실시간 작업 진행상태 표시
- 생성된 산출물 미리보기
- 표준 문서 업로드 및 관리
- 사이트별 설정 관리

**기술 스택**:
- React/Vue.js + TypeScript
- WebSocket for 실시간 업데이트
- 상태 관리: Redux/Zustand
- UI 라이브러리: Material-UI/Ant Design

### 7. File Storage

**역할**: 원본 문서 및 생성된 산출물 저장

**주요 기능**:
- 원본 표준 문서 저장
- 생성된 산출물 저장
- 버전 관리 및 백업
- 접근 권한 제어

**기술 스택**:
- 로컬 파일 시스템 또는 클라우드 스토리지
- S3, Google Cloud Storage, Azure Blob Storage

## 데이터 플로우

### 1. 표준 문서 업로드 플로우

```
사용자 업로드 → Document Processor → Vector DB
     ↓                    ↓               ↓
File Storage ← 원본 파일 저장 ← 메타데이터 ← 임베딩 저장
```

### 2. 산출물 생성 플로우

```
MCP 명령어 → Jira API → 요구사항 추출
     ↓            ↓           ↓
사이트 식별 → Vector DB 검색 → 관련 표준 검색
     ↓            ↓           ↓
RAG Engine → 프롬프트 생성 → LLM 호출 → 산출물 생성
     ↓            ↓           ↓           ↓
File Storage ← 결과 저장 ← 후처리 ← 품질 검증
     ↓
사용자/개발도구 전달
```

## 사용 시나리오

### 기본 사용법

1. **연결 설정**
   ```
   사용자 → Cursor/Copilot → Figure-MCP 연결
   ```

2. **명령어 실행**
   ```
   use design figure {sitename} {jira-ticketId}
   예시: use design figure ecommerce PROJ-123
   ```

3. **자동 처리**
   - Jira 티켓 PROJ-123에서 요구사항 추출
   - ecommerce 사이트의 개발 표준 검색
   - RAG 기반 산출물 생성

4. **결과 전달**
   - 생성된 코드/문서를 Cursor/Copilot으로 전달
   - 또는 지정된 저장소에 업로드

### 관리자 시나리오

1. **표준 문서 관리**
   - Web UI를 통한 표준 문서 업로드
   - 사이트별 분류 및 태깅
   - 버전 관리 및 승인 프로세스

2. **모니터링**
   - 실시간 작업 진행상태 확인
   - 생성 품질 지표 모니터링
   - 사용 통계 및 분석

### 보안 고려사항

- **인증/인가**: OAuth 2.0, JWT 토큰 기반
- **데이터 암호화**: 전송 및 저장 시 암호화
- **접근 제어**: 사이트별 권한 관리
- **감사 로그**: 모든 작업 추적 및 로깅

### 모니터링 및 로깅

- **애플리케이션 모니터링**: Prometheus + Grafana
- **로그 관리**: ELK Stack (Elasticsearch, Logstash, Kibana)
- **성능 메트릭**: 응답시간, 처리량, 오류율
- **비즈니스 메트릭**: 생성 성공률, 사용자 만족도
